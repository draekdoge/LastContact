# Спека: реген энергии кликера — временный инвайт + сеть только «онлайн»

## Назначение

- После **нового реферала по `/start`** пригласитель получает **временное** ускорение регенерации на окно **`invite_regen_boost_hours`** (по умолчанию 5 ч), величина **`invite_regen_boost_bps`**. Окно продлевается от `max(текущий конец, сейчас) + hours`, без стака величины bps.
- **Сетевой** бонус (прямые и косвенные рефералы) считается **только пока они в окне онлайна Mini App**: `mini_app_last_seen_at > now - mini_app_online_presence_seconds` (по умолчанию 180 с).

Поле `users.clicker_regen_bonus_bps` (старые постоянные bps с инвайтов) **не участвует** в формуле эффективного регена.

## Данные

| Поле | Назначение |
|------|------------|
| `mini_app_first_seen_at` | Первый успешный заход (TMA), без влияния на расчёт скорости |
| `mini_app_last_seen_at` | Heartbeat при каждом вызове `sync_mini_app_engagement_and_presence` |
| `invite_regen_boost_ends_at` | До какого момента действует временный bps от инвайта |
| `mini_regen_online_direct` / `mini_regen_online_subtree` | Кэш пересчёта для владельца (прямые / все потомки в онлайне) |

## Формула

- `effective_bps = min(invite_temp_bps + network_bps, MAX_TOTAL_REGEN_BPS)`
- `invite_temp_bps`: `invite_regen_boost_bps` из настроек, если `now < invite_regen_boost_ends_at`, иначе 0
- `network_bps`: как раньше по числу **онлайн** прямых и косвенных (косвенные с множителем 1.5 на сумму bps косвенных), cap `MAX_NETWORK_REGEN_BPS`

## Точки синхронизации онлайна

`sync_mini_app_engagement_and_presence`: вход в Mini App (в т.ч. `login`, `state`, кликер, economy state, tap) — обновляет `last_seen`, пересчитывает кэши у пользователя и всех предков.

## Риски и побочные эффекты

- Частые запросы mini API дают много SQL (рекурсивный подсчёт + обход предков); глубина цепочки обычно мала.
- Параметр `mini_app_online_presence_seconds` сильно влияет на «узкое» окно онлайна; слишком большое — почти постоянный бонус, слишком малое — бонус пропадает между тапами.
- Кэш на строке пользователя может **отставать на секунды** от реального состояния дерева до следующего heartbeat любого узла в поддереве.
