# Proposal: Mini App Clicker — процесс идентификации (v1)

## Статус

Draft v1.0 · новая фича Mini App, расширяет Лабораторию из `lab-fountain-zombie-v1`. Область: короткая активность в Mini App, энергия, мультитач, награды реагентами, UI-анимации. Монетизация через коины/Stars — вне v1, только предусмотрена как будущий источник восстановления энергии.

## 0. Контекст

Лаборатория уже закрывает проблему «контакты закончились, но таймер идёт». Кликер добавляет к ней активное действие: игрок не просто ждёт анализа, а вручную участвует в идентификации штамма и получает реагенты для Фонтана/Лабы.

Ключевой принцип: кликер удерживает игрока в Mini App 5–10 минут, но не заменяет реферальную механику и не продлевает таймер напрямую.

## 1. Глоссарий (спек → код)

| Термин | Код / БД (рабочее имя) | Описание |
|---|---|---|
| Процесс идентификации | `clicker` | Экран кликера внутри вкладки «Лаба» |
| Энергия | `users.clicker_energy` | Расходуемый ресурс для принятых сервером кликов |
| Время обновления энергии | `users.clicker_energy_updated_at` | Точка расчёта пассивного восстановления |
| Прогресс идентификации | `users.clicker_progress` | Счётчик принятых кликов до reward roll |
| Дневные reward roll | `users.clicker_daily_rolls` | Лимит наград за UTC-сутки |
| День лимита | `users.clicker_daily_reset_date` | UTC-дата текущего лимита |
| Реагенты | `users.reagent_dna/rna/cat` | Уже существующие ресурсы Фонтана |

## 2. Вход в механику

Во вкладке Mini App «Лаба» добавляется кнопка:

```text
🧬 Процесс идентификации
Энергия: 840 / 1200
```

Если энергии нет:

```text
🧬 Процесс идентификации
Энергия восстановится через 00:42
```

Кнопка открывает отдельный экран Mini App clicker. В v1 экран доступен всем зарегистрированным игрокам, у которых есть доступ к Mini App. Состояние `infected/zombie/immune` не блокирует вход, но награды и будущие бонусы баланса могут зависеть от состояния.

## 3. UI экрана

### 3.1 Верхний блок

- Полоса энергии сверху.
- Число энергии: `840 / 1200`.
- Таймер до следующей единицы энергии, если энергия не полная.

### 3.2 Центр

- Кликабельный объект: ядро штамма / молекула / капсула.
- Мультитач разрешён: несколько пальцев могут создавать несколько визуальных tap-событий.
- Визуальный tap не равен серверной награде; сервер принимает только клики в рамках энергии и лимитов.

### 3.3 Низ

- Счётчики реагентов: `DNA`, `RNA`, `CAT`.
- Прогресс идентификации до следующего reward roll.
- Короткий лог последней награды: `+1 DNA`, `+1 RNA`, `Образец стабилизирован`.

## 4. Анимации

Минимальный набор v1:

- **Tap burst:** при касании из точки клика появляется маленькая DNA/RNA-частица.
- **Collapse:** частица схлопывается в центр ядра или улетает в счётчик реагента.
- **Energy drain:** верхняя полоска энергии плавно уменьшается после серверного подтверждения.
- **Reward pulse:** при выпадении реагента соответствующий счётчик вспыхивает.
- **Combo glow:** при серии кликов ядро усиливает свечение.
- **Empty energy shake:** если энергии нет, ядро слегка трясётся, а полоска энергии подсвечивается.

Фронт может показывать визуальные эффекты на все касания, но монетизируемые эффекты (`+DNA/RNA/CAT`) показывает только по ответу сервера.

## 5. Баланс v1

| Параметр | Значение v1 | Обоснование |
|---|---:|---|
| `max_energy` | 1200 | Даёт длинную сессию и пространство для мультитача |
| `tap_cost` | 1 | Один принятый сервером tap списывает 1 энергию |
| `batch_window_ms` | 300 | Фронт копит мультитач-события и отправляет пачкой |
| `max_taps_per_request` | 12 | Защита от огромных payload |
| `max_accepted_taps_per_second` | 4 | Экономика не зависит от скорости автокликера |
| `reward_roll_every` | 20 accepted taps | Награда не на каждый клик, но часто ощущается |
| `energy_regen` | +1 / 6 sec | Полное восстановление ≈ 120 минут |
| `daily_reward_roll_cap` | 180 | До 3600 принятых кликов с наградами в UTC-сутки |

Ожидаемая активная сессия:

- При спокойном темпе и анимациях: 7–10 минут.
- При мультитач-спаме: сервер всё равно принимает максимум 4 tap/sec, энергия не сгорает мгновенно.

## 6. Награды

Reward roll происходит каждые 20 принятых сервером кликов.

| Результат roll | Шанс | Награда |
|---|---:|---|
| DNA | 50% | `+1 DNA` |
| RNA | 14% | `+1 RNA` |
| CAT | 1.5% | `+1 CAT` |
| Пустой/стабилизация | 34.5% | Без реагента, только прогресс/анимация |

Гарантии:

- Каждые 100 принятых кликов: гарантированный `+1 DNA`, если за этот отрезок не было реагентов.
- `CAT` не гарантируется, чтобы не обесценить Фонтан.

Критический режим таймера можно добавить позже как балансный модификатор. В v1 не включать, чтобы не смешивать кликер с продлением таймера.

## 7. API

### `GET /api/mini/clicker/state`

Возвращает состояние кликера:

```json
{
  "energy": 840,
  "max_energy": 1200,
  "next_energy_at": "2026-04-29T17:10:06Z",
  "progress": 13,
  "reward_roll_every": 20,
  "daily_rolls": 42,
  "daily_roll_cap": 180,
  "reagents": {
    "dna": 14,
    "rna": 3,
    "cat": 0
  }
}
```

### `POST /api/mini/clicker/tap`

Запрос:

```json
{
  "taps": 8
}
```

Сервер:

1. Проверяет Mini App `initData`.
2. Восстанавливает энергию по `clicker_energy_updated_at`.
3. Сбрасывает дневной лимит, если начался новый UTC-день.
4. Считает `accepted_taps = min(requested, energy, max_taps_per_request, rate_limit_window)`.
5. Списывает энергию только за `accepted_taps`.
6. Увеличивает `clicker_progress`.
7. За каждые `reward_roll_every` прогресса выполняет reward roll, если дневной cap не исчерпан.
8. Начисляет реагенты на сервере.

Ответ:

```json
{
  "accepted_taps": 3,
  "rejected_taps": 5,
  "energy": 837,
  "max_energy": 1200,
  "next_energy_at": "2026-04-29T17:10:06Z",
  "progress": 16,
  "reward": {
    "type": "dna",
    "amount": 1
  },
  "reagents": {
    "dna": 15,
    "rna": 3,
    "cat": 0
  }
}
```

## 8. Антиабуз

Главная защита — энергия. Дополнительная защита нужна только чтобы автокликер не сжигал экономику быстрее честного пользователя:

- Сервер не доверяет фронту и принимает только количество tap-событий.
- Redis rate limit: `clicker:{telegram_id}:tap_window`.
- Максимум 4 принятых tap/sec.
- Максимум 12 tap в одном запросе.
- Награды ограничены дневным cap.
- Все награды начисляются только сервером.

Отклонённые taps могут возвращаться как `rejected_taps`, чтобы фронт не показывал ложные награды.

## 9. Влияние на Лабу и Фонтан

Кликер не продлевает таймер напрямую.

Реагенты используются в существующих механиках:

- `DNA/RNA/CAT` идут в Фонтан.
- Будущая версия может использовать реагенты для ускорения/улучшения Лабы.

Кнопка «Процесс идентификации» живёт во вкладке «Лаба», потому что нарративно это часть анализа образца.

## 10. Изменения схемы БД (дельта)

Минимальный вариант — поля в `users`:

| Поле | Тип | Default |
|---|---|---|
| `clicker_energy` | INTEGER | 1200 |
| `clicker_energy_updated_at` | TIMESTAMPTZ | `now()` |
| `clicker_progress` | INTEGER | 0 |
| `clicker_daily_rolls` | INTEGER | 0 |
| `clicker_daily_reset_date` | DATE | `CURRENT_DATE` |

Отдельная таблица событий в v1 не обязательна. Для аналитики можно добавить позже `clicker_events`.

## 11. Acceptance Criteria

- **AC-CLK-01** Во вкладке «Лаба» есть кнопка `🧬 Процесс идентификации`.
- **AC-CLK-02** Кнопка открывает экран кликера без перезагрузки Mini App.
- **AC-CLK-03** На экране сверху видна энергия `current / max` и прогресс-бар.
- **AC-CLK-04** Tap по центральному объекту создаёт визуальную DNA/RNA-анимацию.
- **AC-CLK-05** Фронт отправляет tap-события батчами, поддерживая мультитач.
- **AC-CLK-06** Сервер принимает не больше 4 tap/sec и не больше 12 tap/request.
- **AC-CLK-07** Энергия списывается только за принятые сервером taps.
- **AC-CLK-08** Энергия восстанавливается серверно: `+1 / 6 sec`, максимум 1200.
- **AC-CLK-09** Каждые 20 принятых taps выполняется reward roll, если дневной cap не исчерпан.
- **AC-CLK-10** Награды начисляются только сервером в `users.reagent_dna/rna/cat`.
- **AC-CLK-11** При нулевой энергии taps не начисляют награды, UI показывает empty-energy состояние.
- **AC-CLK-12** Кликер не меняет `timer_ends_at` напрямую.

## 12. Риски и побочные эффекты

- **Обесценивание реагентов:** слишком щедрые награды могут сломать Фонтан. Смягчение: дневной cap и редкий `CAT`.
- **Автокликеры:** энергия ограничивает общий объём, rate limit ограничивает скорость монетизации.
- **Слишком короткая сессия:** мультитач может визуально сжечь энергию быстро. Смягчение: сервер принимает максимум 4 tap/sec, фронт показывает визуал на все касания, но награды только по accepted taps.
- **Разрыв ожиданий:** игрок может видеть много кликов, но мало наград. UI должен объяснять энергию и прогресс до roll.
- **Нагрузка на API:** батчинг 300 ms снижает число запросов; Redis rate limit защищает от спама.
- **Кликер конкурирует с рефералкой:** не даём прямого продления таймера, только реагенты.

## 13. Вне v1

- Восстановление энергии за коины/Stars.
- Платные бустеры наград.
- Таблица аналитики `clicker_events`.
- Критический множитель при низком таймере.
- Лидерборд кликера.
