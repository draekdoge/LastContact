# ADR 002: Таймер, слои спасения (лаборатория, фонтан, узлы) и разделение бот / Mini App

## Статус

Принято

## Контекст

Игроки не должны ощущать тупик, когда личный соцграф исчерпан. Базовый горизонт таймера (порядка суток) слабо согласуется с глобальной аудиторией и разными часовыми поясами. Нужны осмысленные взаимодействия после кризиса и параллельно с таймером — без полной подмены вирусной механики «соло-фармом».

## Решение (продукт)

### 1. Время и объяснение в UI

- Увеличить базовый горизонт таймера до порядка **48–72 ч** (точные значения — в конфиге окружения; возможен **мягкий вход**: стартовое окно + рост от действий).
- В интерфейсе явно показывать **источники продления**: рефералы, ивенты, лаборатория (и при необходимости «сон» / текущие серверные правила), чтобы рост ощущался заслуженным, а не случайным.

### 2. Три слоя спасения

**A. Локально — «Лаборатория»**

- Короткая цепочка шагов (сбор образца → анализ → ожидание тика / N минут).
- Награда: **отсрочка (часы)** или статус **«стабилизация»** вместо мгновенного перехода в деградацию.
- **Лимит:** 1–2 цикла в сутки и/или «топливо» от активности и шаринга, чтобы не вытеснить реферальный слой.

**B. Глобально — «Фонтан жизни» (мировой ивент)**

- Периодическое серверное окно; общий **прогресс «планеты»** от суммарных действий (шаринги, входы в Mini App, явный вклад в ивент).
- При заполнении прогресса — **пакет часов** или **смягчение угрозы** для целевой группы (например, все заражённые или участники ивента).
- Игрок видит **свой вклад** (доля / очки) — связь «я ↔ планета».

**C. Гибрид — «Узел сети»**

- Виртуальные **регионы** (или шарды без реальной гео): активность региона влияет на **баффы/дебаффы** или доступ к усиленной лаборатории.
- Ограниченная **смена узла** (например, раз в сутки) или условие через приглашение из другого узла — лёгкая социальная связь без бесконечного списка друзей.

### 3. Фаза «после таймера»

- Состояние **zombie / деградация** не трактуется как окончательное удаление из игры: предусматривается **восстановление** через лабораторию, глобальный ивент или цепочку взаимодействий (детали реализации — в следующих ADR/задачах).

### 4. План внедрения (фазы)

| Фаза | Содержание |
|------|------------|
| 0 | Конфиг времени; в UI блок «от чего растёт таймер» (в т.ч. плейсхолдеры под ивенты/лабораторию). |
| 1 | Лаборатория: циклы, лимиты, награды, интеграция с API. |
| 2 | Фонтан жизни: окно ивента, глобальный прогресс, вклад игрока. |
| 3 | Узлы сети: регионы, баффы, смена узла. |

## Решение (архитектура): заражение в боте, остальное в Mini App?

**Принцип:** то, что жёстко привязано к **входу в игру и идентичности Telegram**, остаётся в **боте**; то, что требует **богатого UI, прогрессов и пошаговых сцен**, — в **Mini App** с тем же backend и авторизацией (например `initData` → сессия / пользователь).

| Зона | Бот | Mini App |
|------|-----|----------|
| Первый контакт, `/start`, реферальный payload, начисление заражения рефереру | Да | Нет (кроме открытия по ссылке с последующим заходом в бота при необходимости) |
| Пуши: таймер, предупреждения, старт ивента, «лаборатория готова» | Да | Опционально дубли через in-app |
| Шаринг приглашения (native share / подготовленные сообщения) | Да | Да, если удобнее из контекста мини-аппа |
| Лаборатория: шаги, таймеры ожидания, визуальный прогресс | Краткие напоминания / кнопка «открыть лабораторию» | Основной UX |
| Фонтан жизни: глобальный бар, вклад, история ивентов | Уведомление + deep link | Основной UX |
| Узлы сети: карта/список регионов, смена узла | Минимум (статус в одном сообщении) | Основной UX |
| Настройки, длинные тексты правил | Кратко в боте | По желанию полная версия |

**Итог:** **заражение и реферальный контур — через бота** (единственный надёжный путь для `start` payload и привычки пользователей). **Лаборатория, фонтан, узлы — основной сценарий в Mini App**; бот держит **вход, уведомления и быстрые мосты** (`openWebApp`). Так меньше дублирования сложной логики в чате и проще поддерживать единый API.

## Последствия

- Потребуются эндпоинты и состояние в БД для лаборатории, ивентов и узлов (отдельные миграции и задачи по фазам).
- Mini App становится основным местом «глубокой» игры; бот остаётся критичным для онбординга и retention-пушей.
- Документ не задаёт точные числа баланса — их фиксируем в конфиге и при необходимости в ADR по балансу.
