# ADR 001: Очереди, воркеры и таймеры

## Статус

Принято

## Контекст

Нужна масштабируемая обработка событий (заражения, уведомления) и проверка таймеров без блокировки вебхука Telegram.

## Решение

- **RabbitMQ** — брокер для Celery (`CELERY_BROKER_URL`).
- **Celery worker + Beat** — фоновые задачи и периодические проверки (например раз в 60 с заготовка под истечение таймеров).
- **Redis** — кэш, rate limit, в перспективе TTL таймеров; на первом этапе дублируем `timer_ends_at` в PostgreSQL для простоты MVP.
- Вебхук **aiogram** обрабатывает апдейты синхронно с БД; тяжёлая логика выносится в задачи по мере роста.

## Последствия

- Docker Compose поднимает `rabbitmq`, `redis`, `postgres`, `api`, `worker`, `beat`.
- Прод: мониторинг очередей и dead-letter — отдельная задача.
